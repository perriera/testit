History of C++

" In 1979, Bjarne Stroustrup, a Danish computer scientist, began work on "C with Classes", the predecessor to C++.[8] The motivation for creating a new language originated from Stroustrup's experience in programming for his Ph.D. thesis. Stroustrup found that Simula had features that were very helpful for large software development, but the language was too slow for practical use, while BCPL was fast but too low-level to be suitable for large software development. When Stroustrup started working in AT&T Bell Labs, he had the problem of analyzing the UNIX kernel with respect to distributed computing. Remembering his Ph.D. experience, Stroustrup set out to enhance the C language with Simula-like features.[9] C was chosen because it was general-purpose, fast, portable and widely used. As well as C and Simula's influences, other languages also influenced C++, including ALGOL 68, Ada, CLU and ML. " -- Wikipedia/C++

" C with Classes "

If they say "Necessity is the mother of invention.", then C++ is a perfect example of precisely that. You had a man that had obtained a Ph.d in computer science going into a real life situation where theory meets reality. And the reality of AT&T is that there was a lot of C code that could no longer be managed properly. So, stemming from his thesis, he began experimenting with a pre-compiler, one that could take the object-oriented techniques he had learned from Simula 67 and make them possible inside the world of the C programming language. It was a simple technique, simply create some sort of 'class' structure that would be translated into proper C terminology. Hence the phrase "C with Classes" came into being. Once the magic of having a pre-compiler convert what he really wanted to do into standard C code, the race was on to put more and more object-oriented features that could be converted in C. Such features were derived classes, strong typing, inlining, default arguments, virtual functions, function name and operator overloading, references, constants, type-safe free-store memory allocation (new/delete), improved type checking, and BCPL style single-line comments.

In time, the pre-compiler became it's own stand-alone compiler known for C++ called CFront. C++ would acquire it's formal name due to a comment made by Rick Mascitti in 1983, in an effort to describe what C++ was all about with respect to C.

The C++ Programming Language

The first edition of The C++ Programming Language was released in 1985, which became the defacto reference for the language. C++ had it's first commercial implementation of C++ was released in October of the same year. C++ would become mainstream in 1990 with the release of Borland's Turbo C++ which was anticipated after the huge success of their Turbo C product.

The Borland C++ product came with tutorial material and the effects felt by newcomers to the world of C++ over C was staggering. It revolutionized the way things were to be done in the PC software development community especially with engineering firms. Life became a lot easier with Borland's Turbo C++ on your shelf. In time, Microsoft would have a C/C++ compiler but it would take a few years before the Microsoft product could over take the Borland product in terms of quality and ease of use. But then there is a story whereby Borland accidentally laid off their chief C++ compiler architect and Microsoft swooned in on the opportunity to steal him away to the tune of a $1,000,000 signing bonus. He never looked back and soon after Microsoft's C++ compiler (Visual C++) dominated the market.

Other compilers such as GCC and CLang were there for the Unix and Linux platforms, but if you weir part of the PC world then there were really only two choices, Borland or Microsoft when it came to C++. In the Apple world, Xcode today is the only real name when it comes to C++ but they had their initial competitor to C++ known as Objective-C which in time gave way to Swift. Swift differs from C++ such that it provides it's own garbage collection, much like Java and therefore is not like C++ because C++ gives you low level control over memory resources and other benefits necessary when programming at the machine or operating system level.

Why was it designed?

In the beginning many computer manufacturers were developing computer hardware essentially from scratch starting with the UNIVAC from the days of WWII as the forerunner. Each company manufactured their own hardware and developed their own unique software to work with it. This led to a situation where essentially the same software functionality was being re-written for different platforms. Hence the need for code portability came into being as a cost saving measure and of course, a lucrative business endeavour in it of itself.

The early computer languages for operating systems to operate on the hardware took on the form of Machine Language first (which was really just a step above programming a computer using arrays of manual switches and vacuum tubes) and that led to a form of programming called Assembler where instructions could be more easily encoded into Machine Language (which was just a fancy name for programming a computer using only numbers as operational codes and values). Assembler was a just step above this allowing programmers to use letters to represent the codes they were programming instead of pure numbers. However, even the Assembler language was very specific to the different vendors of computer hardware. Hence a need for a way to covert software applications into Assembler independent of the hardware vendor brought about the creation of a language called 'C' compliments Dennis Ritchie between the years of 1969 and 1973 at Bell Labs. In effect, you would write your software business logic in C and the C compiler would convert your C code into Assembler which in turn 'assembled' your code into the 'Machine Language' of a particular computer of a specific computer hardware vendor. In effect, your C code could then be easily ported, without any changes to the C code, to another computer, ideally from another computer hardware vendor and your code would compile into the machine language with no more fuss to it than that. Indeed, the cost saving benefits of this brought on the need for competing computer hardware vendors to develop their own implementation of the C language to match their Assembler language specifics.

Once the ability to write portable code became a popular practise, the need for easier code re-use manifested itself, hence the need to write code in such a way as to make use of code that has already been written earlier for other purposes. Bjarne Stroustrup, with a background in Simula, developed in the 1960’s, was one of the first to experiment with applying some of the techniques used in that experimental “object-oriented” language with C, creating a form of C known as " C with Classes ". Which in time, because known as C++ where an emphasis on operating system software where performance, efficiency and flexibility were key. However, C++ soon found itself as the choice language for just about every type of software application. Nevertheless, when it came to infrastructure and resource-constrained applications, including desktop applications, e-commerce, web servers and performance-critical applications (e.g. telephone switches, space probes, military or medical) C++ was often the preferred language of choice. Both open source (Free Software Foundation's GCC. LLVM) and commercial companies (Microsoft, Intel and IBM) specialize in C++ compiler tools.
